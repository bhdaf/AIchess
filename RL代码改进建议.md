# AIchess 代码改进建议（基于 run_RL 真实日志）

日期：2026-03-26  
适用范围：`AIchess/runs/run_RL`、`AIchess/train.py`、`AIchess/opponent_pool.py`、`AIchess/mcts.py`

## 一、结论先行（为什么当前效果不好）
- 当前 200 局 RL 总得分（按我方视角）仅 `0.3925`（52 胜 / 95 负 / 53 和）。
- 对 `pikafish_weak/mid/full` 合计 `50` 局全部未胜（0 胜）。
- 评测仅每 50 局、每次 20 盘，方差很大：`0.50 -> 0.75 -> 0.25 -> 0.50`。
- 有效训练样本仅 `6156`（`self_play.jsonl` 生成 10506，和棋过滤后只入 buffer 6156），对于 256 通道、16 残差块模型明显偏少。

## 二、优先级 P0（建议先做）

### P0-1. 扩大且稳定评测，避免“误判提升/退化”
- 建议改动文件：`train.py`
- 现状问题：`eval_games=20` 太小，且只评测 `previous`，统计噪声高。
- 建议：
  - `eval_games` 提升到 `60~100`。
  - 增加固定基准对手评测（如 `self_play` + `historical` + `pikafish_weak` 三路）。
  - 日志里记录每路对手分数和总分，避免单一指标波动。

### P0-2. 调整课程学习节奏，减少中后期“被强对手压制”
- 建议改动文件：`opponent_pool.py`
- 现状问题：后 1/3 训练里 `pikafish_mid/full` 提前占比较高，且当前模型尚弱，导致负样本密集。
- 建议：
  - 后 1/3 阶段先不引入 `pikafish_full`（或降到 0.05 以下）。
  - 先让 `self_play + historical + weak` 达到稳定后再放开 mid/full。
  - 加入基于胜率的自适应课程（低于阈值自动回退到更弱课程）。

### P0-3. 放宽和棋过滤，保留更多可学信号
- 建议改动文件：`train.py`
- 现状问题：`maxmoves/repetition` 和棋保留概率都接近 0.05，本次仅保留了 2 局和棋样本。
- 建议：
  - 将 `keep_draw_maxmoves_prob` 调高到 `0.20~0.35`。
  - 将 `keep_draw_repetition_prob` 调高到 `0.10~0.20`。
  - 按训练阶段动态调整（前期高、后期低），而非固定常数。

## 三、优先级 P1（第二批）

### P1-1. 训练方式改为“采样小批次更新”，避免每局全量重训
- 建议改动文件：`train.py`（`train_model` 调用策略）
- 现状问题：每局都对整个 `data_buffer` 训练多轮，计算开销大且容易放大旧数据影响。
- 建议：
  - 每局只采样固定步数（如 1k~4k 样本）做若干梯度更新。
  - 增加“近期样本优先”策略（例如 recent:old = 7:3）。
  - 将训练触发从“每局都训”改为“每 N 局训一次，但每次更集中”。

### P1-2. 提升样本吞吐：并行自对弈 + 推理批处理
- 建议改动文件：`train.py`、`inference_server.py`
- 现状问题：`num_selfplay_workers=0`，单进程 CPU 导致样本增长慢。
- 建议：
  - 有 GPU 条件时启用 `num_selfplay_workers=4~8`。
  - 开启 `bf16/tf32`（硬件支持时）并调优 `inference_batch_size_max`。
  - 没有 GPU 时，降低网络规模（临时训练模型）换取更多对局数。

### P1-3. 加入“训练健康度监控”与早停保护
- 建议改动文件：`train.py`、`export.py`
- 建议新增监控项：
  - 按对手类型的我方胜率/得分率。
  - 近 20 局 loss 均值、方差、斜率。
  - 和棋类型分布（maxmoves/repetition/other）与保留率。
- 触发条件示例：若连续两次评测下降且对 weak 对手胜率为 0，则自动回退课程或降学习率。

## 四、优先级 P2（结构优化）

### P2-1. 探索噪声退火
- 建议改动文件：`mcts.py`、`train.py`
- 现状问题：全程固定 Dirichlet 噪声权重，后期可能引入过多随机性。
- 建议：随训练进度降低 `dirichlet_weight`，例如从 `0.25` 逐步退火到 `0.05`。

### P2-2. 学习率和损失权重调度
- 建议改动文件：`train.py`
- 建议：
  - 使用余弦退火或分段降学习率。
  - 分离 policy/value loss 权重并可配置，避免 value 过强牵引。

### P2-3. 启动权重路径强校验（防止误用随机初始化）
- 建议改动文件：`train.py`
- 现状风险：`init_from_distill` 路径不存在时会静默走其他分支，可能非预期。
- 建议：
  - 增加 `--strict_init_from_distill`。
  - 路径不存在时直接报错退出。

## 五、建议的下一轮最小可行实验（不改模型结构）
1. 对局数从 `200` 提升到 `1000+`。  
2. 评测改为每 `100` 局、每次 `80` 盘。  
3. 课程学习先禁用 `pikafish_full`。  
4. 和棋保留率提高到 `maxmoves=0.25`、`repetition=0.15`。  
5. 学习率从 `2e-4` 改为分段：`2e-4 -> 1e-4 -> 5e-5`。  
6. 每轮结束输出“按对手类型”的胜率表，用于论文直接引用。
